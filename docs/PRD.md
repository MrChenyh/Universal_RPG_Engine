# Universal RPG Engine PRD

## 1. 产品概述

Universal RPG Engine（万界剧情模拟引擎）是一个面向 OpenClaw/ClawX 的文字 RPG 技能。它允许用户上传小说文本或原创大纲，并将原始文本转化为可游玩的沉浸式剧情模拟体验。

产品核心目标是让用户无需手动整理世界观、角色表和剧情节点，就能基于任意 `.txt` 或 `.md` 剧本快速启动文字 RPG。系统需要完成长文本切片、剧情索引、角色创建、难度选择、回合叙事、玩家档案同步等流程。

## 2. 背景与问题

长篇小说或复杂剧本直接进入大模型上下文时，容易出现以下问题：

- 文本过长导致 Token 成本高、上下文溢出或处理失败。
- 剧情推进缺少稳定索引，模型容易脱离原著时间线。
- 玩家属性、背包、外挂状态和 NPC 好感度如果反复展示，会让聊天界面臃肿。
- 传统文字冒险通常缺少难度分级，难以兼顾爽文体验和高压挑战体验。

Universal RPG Engine 通过剧情切片、索引映射、难度模式和云端档案托管解决这些问题。

## 3. 目标用户

- 希望把小说、网文或原创大纲改造成文字 RPG 的玩家。
- 希望用 OpenClaw/ClawX 快速搭建互动剧情技能的创作者。
- 希望保留原著时间线，同时允许自定义行动和角色成长的沉浸式文本游戏用户。

## 4. 产品目标

- 支持用户上传 `.txt` 或 `.md` 剧本文本，并自动生成剧情切片与索引。
- 支持百万字级别长篇文本的分段处理，降低单次上下文压力。
- 提供爽文、普通、苦难三种命运模式。
- 提供角色创建流程，并保留自定义开局能力。
- 每回合输出沉浸式叙事和 A/B/C/D 四个行动选项，其中 D 固定为自定义行动。
- 使用飞书文档或表格保存玩家档案，让聊天界面保持清爽。

## 5. 核心功能

### 5.1 剧本导入与切片

用户上传 `.txt` 或 `.md` 文件后，系统读取文件内容，并按固定大小拆分为多个剧情数据包。

当前实现默认以约 120000 字符为一个切片单位，输出到 `game_data/` 目录，并生成 `index_map.json`。

系统需要识别常见章节标题，例如“第十章”“第 1 卷”“第003回”等，并在索引中记录每个切片覆盖的章节范围。

### 5.2 剧情索引

系统生成的 `index_map.json` 用于帮助 DM 根据当前剧情阶段读取对应切片。

索引需要包含：

- 切片文件名。
- 该切片覆盖的起始章节与结束章节。

### 5.3 角色创建

当用户输入“开始游戏”时，系统启动初始化引导。

角色创建需要包含：

- 内置身份选项。
- 自定义开局选项。
- 初始特长或核心能力设定。
- 与所选命运模式相匹配的初始资源。

### 5.4 命运模式

系统提供三种难度：

- 爽文模式：自动开启专属系统或面板，机缘更高，默认不触发永久死亡。
- 普通模式：遵循原著逻辑，无外挂，存在死亡判定。
- 苦难模式：资源稀缺、风险更高、死亡概率更高。

### 5.5 回合交互

每一回合必须包含：

- 第一人称沉浸式剧情描写。
- 与当前剧情阶段一致的事件推进。
- A/B/C/D 四个行动选项。
- D 选项固定为“自定义行动”。

系统应尽量避免脱离原著剧情时间线进行无依据扩写。

### 5.6 飞书玩家档案

游戏初始化时，系统应创建《万界玩家档案》，用于记录：

- 玩家基础属性。
- 背包物品。
- 外挂或系统状态。
- NPC 好感度。
- 当前剧情位置。
- 重要任务与事件标记。

当玩家获得物品、属性提升或 NPC 好感变化时，系统应更新飞书档案。

聊天回复末尾只保留飞书档案链接和简短变化摘要，避免每回合重复输出完整面板。

## 6. 用户流程

1. 用户安装并启用 Universal RPG Engine 技能。
2. 用户上传小说文本或原创大纲。
3. 系统进行剧情切片并生成索引。
4. 系统提示用户选择命运模式。
5. 用户创建角色或选择自定义开局。
6. 系统创建飞书玩家档案。
7. 游戏进入回合制叙事。
8. 用户通过 A/B/C 选项或 D 自定义行动推动剧情。
9. 系统持续同步玩家档案与剧情状态。

## 7. 非功能需求

- 稳定性：处理大文件时应避免重复切片已有数据。
- 可读性：聊天界面应保持简洁，不输出冗长属性面板。
- 可扩展性：后续可以接入更多云端存档方式或更多章节识别规则。
- 可维护性：切片逻辑、索引生成和游戏运行指令应保持清晰分离。
- 兼容性：优先支持 OpenClaw/ClawX 技能安装与运行方式。

## 8. 成功指标

- 用户上传 `.txt` 或 `.md` 后，可以成功生成 `game_data/` 与 `index_map.json`。
- 用户能在三种命运模式中选择并进入游戏。
- 每回合均能提供四个行动选项，且 D 为自定义行动。
- 玩家档案能被创建并在关键状态变化时更新。
- 长篇文本游玩时，剧情推进能参考索引并减少脱离原著的问题。

## 9. 里程碑

### M1：基础仓库与技能声明

- 完成 `README.md`。
- 完成 `SKILL.md`。
- 完成基础脚本入口。

### M2：文本切片与索引

- 支持 `.txt` 与 `.md` 文件导入。
- 输出 `game_data/chunk_*.txt`。
- 输出 `game_data/index_map.json`。

### M3：游戏流程

- 支持命运模式选择。
- 支持角色创建与自定义开局。
- 支持标准回合输出规范。

### M4：飞书档案

- 创建《万界玩家档案》。
- 同步属性、背包、外挂状态和 NPC 好感度。
- 在聊天末尾输出档案链接与简短变化摘要。

### M5：体验优化

- 优化章节识别。
- 避免重复处理已切片数据。
- 增加异常提示与恢复流程。

## 10. 风险与待确认事项

- 飞书 API 或 OpenClaw 工具调用能力需要在真实运行环境中确认。
- 当前章节识别主要覆盖中文章节标题，其他格式需要后续扩展。
- 大文件读取与切片需要关注内存占用。
- 剧情索引只能辅助定位，仍需要 DM 指令约束模型避免偏离原著。
